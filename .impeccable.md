# 디자인 컨텍스트 — 농산물 판매장부

## 대상 사용자
- 사용자의 어머니 (고령의 농부). 스마트폰(안드로이드 크롬)으로 사용.
- IT 익숙하지 않음. 밭·시장 등 야외에서 사용할 수 있음.

## 핵심 용도
- 판매처(가게)별로 농산물을 **내놓음 / 판매 / 회수** 기록
- 판매처별 현황(남은 수량)과 지난 기록을 한눈에 확인
- 데이터는 아들(sh9.lee@gmail.com)의 구글시트에 동기화

## 톤 & 원칙
- 따뜻한 종이 장부 느낌: 미색 종이 배경 + 진한 녹색 헤더/잉크, 밝은 라이트 테마 고정 (야외 시인성)
- 큰 글씨(기본 18px+), 큰 터치 버튼(56px+), 한 화면에 한 가지 일
- 어려운 용어·암호 표기 금지: "3—3" 같은 축약 대신 "있던 3 · 놓음 3"처럼 라벨 병기
- 필드 용어: 남아 있던 것(left) / 새로 갖다 놓음(added) / 팔림(sold) / 도로 가져옴·회수(taken)
- 남은 수량은 직접 입력이 아니라 자동 계산 (있던+놓음−팔림−회수), 입력 중 실시간 표시
- 장부는 두 가지 보기: 한눈에 보기(행=가게, 열=작물 표, 칸=남은 개수) / 자세히 보기(가게 카드 + 칩)
- 모달 금지, 두 번 탭으로 삭제 확인
- 색만으로 구분하지 않기 (항상 글자 병기)

## 기술 제약
- 순수 HTML/CSS/JS (프레임워크 없음), GitHub Pages 정적 호스팅
- 데이터: localStorage 캐시 + Google Apps Script(구글시트) 동기화
- 오프라인에서도 열리고 기록 가능해야 함 (서비스 워커 + 대기열)
