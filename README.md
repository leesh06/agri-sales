# 농산물 판매장부

엄마가 폰으로 쓰는 농산물 위탁판매 기록 웹앱.
메모장에 손으로 쓰던 방식 그대로 — 가게(세로) × 작물(가로) 표에서 칸을 눌러
**남은 것 — 새로 놓음 — 팔림** 세 숫자를 적는다.

- 화면은 항상 폰에 저장된 데이터로 즉시 표시 (딜레이 없음)
- 기록은 뒤에서 자동으로 구글시트에 전송 (인터넷 끊겨도 나중에 자동 전송)
- 크롬에서 "홈 화면에 추가"하면 일반 앱처럼 사용 가능 (오프라인에서도 열림)

## 구성

```
index.html          앱 화면
css/style.css       디자인
js/app.js           기능 (장부/지난기록/관리 + 시트 동기화)
manifest.json       홈 화면 설치(PWA) 설정
sw.js               오프라인 지원 서비스 워커
icons/              앱 아이콘
apps-script/Code.gs 구글시트에 붙이는 연동 스크립트
```

- 데이터 시트: 드라이브의 **농산물 판매장부** (ID: `1XjLPMHd9E1rNTOo_daVuQ2fTJ0uoJvOtUgb7Y2DxM5U`)
  - `판매기록` 탭: id | 날짜 | 판매처 | 품목 | 남음 | 새놓음 | 팔림 | 기록시각
  - `설정` 탭: A열 판매처 목록, B열 품목 목록
  - 집 잔량은 판매처 이름 `집`으로 저장됨

## 설정 순서 (한 번만 하면 됨)

### 1. 구글시트에 스크립트 붙이기

1. 드라이브에서 **농산물 판매장부** 시트 열기
2. 메뉴 [확장 프로그램] → [Apps Script] 클릭
3. 편집기에 기본으로 있는 코드를 지우고, `apps-script/Code.gs` 내용을 통째로 붙여넣기
4. 저장(💾) 후 오른쪽 위 [배포] → [새 배포] 클릭
5. 유형 선택(⚙️)에서 **웹 앱** 선택
6. 설정:
   - 실행 계정: **나(sh9.lee@gmail.com)**
   - 액세스 권한: **모든 사용자**
7. [배포] 클릭 → 권한 승인 (처음엔 "확인되지 않은 앱" 경고가 떠도 [고급] → [이동]으로 승인)
8. 나오는 **웹 앱 URL** 복사 (`https://script.google.com/macros/s/…/exec` 모양)

### 2. 앱에 주소 연결

방법 A — 앱의 [관리] 탭 → "구글시트 연결"에 URL 붙여넣고 [주소 저장]
방법 B — `js/app.js` 맨 위 `DEFAULT_SCRIPT_URL`에 URL을 넣고 배포 (엄마 폰에서 설정할 필요 없어짐, 추천)

### 3. GitHub Pages 배포

1. GitHub에 저장소 만들고 이 폴더 푸시
2. 저장소 [Settings] → [Pages] → Branch: `main`, 폴더 `/` 저장
3. 몇 분 뒤 `https://<계정>.github.io/<저장소>/` 주소가 생김

### 4. 엄마 폰에 설치

1. 엄마 폰 크롬에서 위 주소 열기
2. 메뉴(⋮) → **홈 화면에 추가**
3. 이후 홈 화면의 `판매장부` 아이콘으로 실행

## 참고

- 스크립트를 수정하면 [배포] → [배포 관리]에서 **기존 배포를 수정**해야 URL이 유지됨
  (새 배포를 만들면 URL이 바뀌어서 앱에 다시 등록해야 함)
- 시트를 직접 고쳐도 됨 — 앱이 다음 동기화 때 시트 내용을 읽어옴
- 앱 파일을 고치면 브라우저 캐시(서비스 워커) 때문에 두 번째 새로고침에 반영될 수 있음
